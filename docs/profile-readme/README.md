I turn high-stakes, complicated software into products people can actually use. Ten years across architecture, product design, and design systems taught me the same craft in two mediums: **how structure makes complex things usable.** First in buildings, now in enterprise software.

> **Complexity isn't the enemy. Confusion is.** The hardest, most valuable design work isn't adding beauty — it's imposing structure: finding the system, naming its parts, and revealing the one path that makes sense.
